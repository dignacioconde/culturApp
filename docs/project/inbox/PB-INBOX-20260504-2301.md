# REDISEÑO UI v3 — Dirección visual completa de CachésApp

REDISEÑO UI v3 — Dirección visual completa de CachésApp

## Contexto
Documento de misión completo para rediseñar la UI de Cachés/Culturapp usando redesign-v2.html y redesign-v3.html como referencia. La versión preferida es v3 (sidebar más cálido, mejor contraste, drawer lateral de detalle, mejor separación navegación/contenido/detalle).

---

## Identidad de producto

La app NO debe parecer: SaaS corporativo frío, dashboard genérico azul/morado, CRM enterprise, app bancaria, panel startup con gradientes.

Debe sentirse: cálida, editorial, clara, cultural, artesanal, profesional sin ser corporativa, simple pero no pobre.

> 'Esta herramienta entiende la vida real de un profesional cultural: fechas, cachés, cobros, gastos, ensayos, clientes, proyectos y bolos.'

## Drawer de detalle de bolo (patrón v3)

Desktop: panel derecho ~380px. Mobile: full-screen o bottom sheet.
Secciones: header (color de proyecto, título, cliente, fecha, estado, botón cerrar) → KPIs (bruto/neto) → Ingresos → Gastos → Info del bolo → Footer (Editar / Ver detalle / Marcar cobrado / Añadir gasto).
Overlay con blur. Cierre con Escape y click fuera. Accesible con teclado.

## Pantallas a rediseñar

1. **Dashboard/Inicio**: saludo editorial, selector de mes, segment control (Cobros/Proyectos), KPIs (ingresos previstos/cobrados, gastos, caché bruto/hora, próximos bolos), panel cobros pendientes, panel próximos bolos.
   - KPIs desktop: grid de 5. Medio: 2 cols. Mobile: 2 cols.
   - Barra superior de color sutil en cada KPI.
   - Fuente mono para cantidades, labels uppercase pequeños.

2. **Agenda semanal**: días como columnas, horas como filas, eventos como bloques coloreados, línea de hora actual, leyenda de proyectos, toggles de capas (Bolos/Ensayos/Tareas/Cobros), bandas de proyecto. Celdas con líneas finas dashed. Día actual con círculo rojo/papel.

3. **Bolos**: lista con fecha, mes, nombre, cliente, hora, estado, importe, color de proyecto. Click → drawer en desktop, full-screen en mobile.
   Estados: confirmed → Confirmado / draft → Borrador / in_progress → En curso / completed → Completado / cancelled → Cancelado.

4. **Ingresos**: tabla limpia con concepto, proyecto, fecha, bruto, IRPF, neto, estado. Badge Cobrado/Pendiente. Fuente mono para cantidades. Header uppercase.

5. **Mobile**: header sticky, marca pequeña, título de página, tab bar inferior (Inicio/Agenda/Bolos/€), FAB acción principal. Agenda en week strip, listas compactas.

## Plan de implementación (8 fases)

1. Auditoría del proyecto actual
2. Tokens centralizados
3. Layout base (AppShell, Sidebar, TopBar, MobileTabBar)
4. Componentes base (Button, KpiCard, Panel, StatusPill, PaidBadge, MonthNavigator, SegmentControl, EmptyState)
5. Pantallas (Dashboard, Agenda, Bolos, Ingresos)
6. Drawer (BoloDetailDrawer, accesibilidad, responsive)
7. Mobile (tab bar, FAB, agenda mobile, listas compactas)
8. QA visual y técnico

---

## Reglas de implementación

NO: pegar HTML del prototipo, estilos inline sin justificación, componentes >700 líneas, colores fuera de tokens, negro puro en sidebar, azul corporativo como primario, librerías UI pesadas sin aprobación.

SÍ: tokens centralizados, componentes reutilizables, props limpias, datos reales, estados vacíos, loading/error, responsive real, accesibilidad básica.

---

## Criterios de aceptación visuales

- Fondo paper, sidebar cálido oscuro, rojo terracota como acción principal
- Serif editorial en marca/títulos, mono para importes/fechas
- Cards con borde sutil, KPIs consistentes, pills consistentes
- Tablas limpias, calendario legible
- Mobile con tab bar inferior y FAB
- Tokens centralizados, no hardcodeados
- No se rompe funcionalidad existente
