# ADR — profiles.tax_rate es la fuente canónica de IRPF; no...

ADR — profiles.tax_rate es la fuente canónica de IRPF; no usar auth metadata

Decisión: el IRPF por defecto en formularios de ingresos (eventos y proyectos) y en settings debe venir de profiles.tax_rate vía useProfile. No usar user.user_metadata.tax_rate ni ningún otro origen.

Motivación: los formularios de ingresos de eventos y proyectos aún usaban user_metadata.tax_rate por defecto. Corregido en commit e7267d4. El error generaba IRPF incorrecto silenciosamente si el usuario había cambiado su tipo en settings.
