# ADR — Protocolo de escritura en AGENT_STATE.md

ADR — Protocolo de escritura en AGENT_STATE.md

Decisión: los agentes SOLO deben añadir su señal/bloque en AGENT_STATE.md, nunca reescribir el archivo completo.

Gotcha detectado (2026-05-04): el agente cultura-lead al cerrar la issue #48 sobrescribió AGENT_STATE.md desde cero, borrando la sección completa 'Estado por agente' (9 bloques de agentes) y señales anteriores. El archivo pasó de 103 líneas a 29.

Forma correcta: leer el archivo primero, añadir solo la entrada nueva en 'Señales activas' y/o actualizar solo el bloque del agente correspondiente en 'Estado por agente'.

Verificación recomendada: tras cada run de agentes que toque AGENT_STATE.md, revisar con git diff HEAD~1 .opencode/AGENT_STATE.md que no se hayan borrado secciones. Si se borraron, restaurar desde git show HEAD~1:.opencode/AGENT_STATE.md
