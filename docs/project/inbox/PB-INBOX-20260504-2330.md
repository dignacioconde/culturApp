# ADR — useProfile como patrón obligatorio; no llamar Supab...

ADR — useProfile como patrón obligatorio; no llamar Supabase directamente desde componentes

Decisión: los componentes no deben llamar a Supabase directamente para leer o escribir el perfil del usuario. Todas las lecturas/escrituras de perfil deben pasar por el hook src/hooks/useProfile.js.

Motivación: Settings.jsx llamaba a Supabase directamente (detectado en commit 9f6e1c). El hook centraliza la lógica, evita inconsistencias y facilita tests.

Extensión: si se necesita un nuevo tipo de dato de perfil, extender useProfile — no crear accesos directos en el componente.
