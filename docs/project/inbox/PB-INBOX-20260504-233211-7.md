# Contexto — Roles de agentes OpenCode: Explorer / Worker /...

Contexto — Roles de agentes OpenCode: Explorer / Worker / Reviewer

El sistema de agentes usa tres roles:
- Explorer: lee código, diagnostica y propone. NO edita archivos.
- Worker: implementa una tarea concreta con ownership explícito de archivos o módulos.
- Reviewer: revisa el diff desde postura de code review — bugs, regresiones, inconsistencias con AGENTS.md y tests faltantes.

Cuándo separar roles: cuando la tarea tenga riesgo o varias áreas afectadas. No hacer investigación manual previa por defecto; lanzar los agentes y dejar que diagnostiquen.

Reportar bugs visuales: incluir ruta, viewport o condición, síntoma/captura y criterio visual de aceptación. Para calendarios, exigir evidencia de toolbar, header y filas del mes visibles.
