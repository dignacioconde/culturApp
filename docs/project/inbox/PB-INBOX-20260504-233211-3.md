# ADR — Controles de formulario custom obligatorios; no usa...

ADR — Controles de formulario custom obligatorios; no usar select/input nativos en páginas

Decisión: las páginas de la app no deben introducir select, input[type=date] ni input[type=datetime-local] nativos. Usar los controles compartidos Select, Input[type=date] e Input[type=datetime-local] de src/components/ui/Input.jsx.

Motivación: en mobile los menús nativos tienen UX deficiente. El trabajo de UX móvil (commit fc99c25) unificó los controles. Si se necesita un nuevo tipo de selector, extender el componente compartido.

UX contract de fechas y decimales:
- Evento start default: 08:00
- End time: mismo día, una hora después del start (eventos de un día por defecto)
- Opción explícita Evento de varios días antes de pedir fecha de fin diferente
- Inputs decimales (IRPF, tax_rate): usar inputMode=decimal + parseDecimal() de src/lib/formatters.js, aceptar coma y punto, validar 0-100
- Controles de fecha: admitir entrada manual DD/MM/YYYY, emitir YYYY-MM-DD a la BD
