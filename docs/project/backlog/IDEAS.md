# Ideas

Entrada ligera para posibilidades que aun no tienen problema, usuario, alcance o release claros.

## Regla de entrada

Las ideas nuevas entran primero por `docs/project/inbox/` mediante `pb:capture`. Despues de curarlas, pueden acabar aqui, en [[TRIAGE]], en una issue `CACH-*`, en una ADR o archivadas.

## Ideas abiertas

| Idea | Estado | Proximo paso |
|---|---|---|
| Pendiente de curar inbox residual | Abierta | Revisar `docs/project/inbox/` en la siguiente curacion. |

## Criterios para convertir en issue

- Hay usuario o flujo afectado.
- Hay problema observable.
- El cambio puede validarse.
- El alcance cabe en una rama de trabajo.
- Puede asignarse a una release o declararse explicitamente fuera de release.
