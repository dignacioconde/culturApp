# 409 en creación de proyecto/evento suele indicar profile faltante

## Idea Principal

Si crear un proyecto o evento devuelve 409 para un usuario autenticado, una causa probable es que falta su fila en `public.profiles`.

## Contexto

`projects.user_id` y `events.user_id` referencian `profiles`. El trigger `handle_new_user` deberia crear la fila al registrarse, pero puede fallar si la funcion no inserta en `public.profiles` explicitamente o si hubo usuarios creados antes de corregir el trigger.

Esto bloqueo a usuarios reales durante el MVP.

## Reparacion

- Verificar que `handle_new_user` inserta en `public.profiles`.
- Buscar usuarios en `auth.users` sin fila correspondiente en `public.profiles`.
- Insertar las filas faltantes despues de confirmar el esquema y RLS.

## Implicaciones

- El onboarding debe detectar o reparar perfil faltante.
- Los errores de creacion no deben mostrarse al usuario como un fallo generico si el origen es recuperable.
