# CACH-0084 — Unificar detalles y formularios de proyectos y eventos

## Objetivo

Eliminar mezcla visual legacy en detalles y formularios de proyectos/eventos sin tocar formulas, normalizadores, hooks ni CRUD.

## Alcance

Incluido:

- Headers, resumentes financieros, tablas/listas, estados vacios, modales, quick actions, confirmaciones visuales, swatches y checkboxes.
- Uso de primitivas y tokens Lovable donde reduzcan duplicacion.

Fuera de alcance:

- Cambiar formulas financieras, agregaciones, `normalizeIncomeForm`, `normalizeExpenseForm`, hooks de datos o rutas.
- Navegacion origin-aware o redisenos funcionales de Work.

## Criterios de aceptacion

- [x] `/projects/:id` y `/events/:id` se perciben dentro del mismo sistema visual que Work/Dashboard.
- [x] Formularios Project/Event mantienen wrappers compartidos para fechas/selects.
- [x] Bottom action bar movil, modales y tablas no solapan ni cortan texto.
- [x] No cambian importes, calculos, estados de cobro ni contratos de datos.

## Resultado

Implementada por worker frontend en `src/pages/Projects/*` y `src/pages/Events/*`: detalles, formularios, modales, tablas/listas, checkboxes, swatches y acciones usan tokens Lovable sin tocar hooks, normalizadores, formulas ni CRUD.
