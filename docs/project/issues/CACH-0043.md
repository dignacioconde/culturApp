# CACH-0043 — [UX] Limpiar acciones en detalle de proyecto

## Objetivo

Hacer que el detalle de proyecto tenga una cabecera mas calmada y acciones mejor jerarquizadas, especialmente en movil.

## Alcance

- Revisar acciones primarias y secundarias en detalle de proyecto.
- Priorizar acciones esperables: editar proyecto, crear evento, anadir ingreso/gasto y eliminar.
- Mover acciones destructivas o raras fuera de la superficie principal.
- Reducir carga visual de cabecera, botones y bloques financieros en movil.

Queda fuera modificar calculos financieros, cambiar contratos de hooks o redisenar el modelo proyecto-evento.

## Criterios de aceptacion

- [x] La accion principal del detalle de proyecto es evidente en movil y desktop.
- [x] Las acciones secundarias no compiten visualmente con el contenido.
- [x] La accion destructiva no queda expuesta como boton dominante.
- [x] La cabecera no ocupa de forma desproporcionada el primer pantallazo movil.
- [x] El flujo existente de edicion sigue funcionando.

## Validacion

- Ejecutar `npm run lint` y `npm run build`.
- Revisar detalle de proyecto con y sin eventos, ingresos y gastos.
- Probar viewports 320, 390, 768 y desktop.
