# CACH-0065 — Ampliar onboarding como tutorial revisitable

## Objetivo

Convertir `/onboarding` en un tutorial mas completo que explique el modelo mental de Cachés y pueda revisarse desde Ajustes.

## Alcance

- Mantener `/onboarding` como ruta obligatoria para usuarios nuevos.
- Explicar trabajos, proyectos, eventos, cobros, gastos, contratantes, exportacion y uso movil.
- Mostrar una version compacta en movil para que el tutorial no fuerce una tarjeta de alto completo.
- Permitir cerrar el tutorial sin obligar a recorrer todos los pasos.
- Mantener el destino final en `/settings`.
- No anadir schema, telemetria ni un sistema de tutorial persistente.

## Criterios de aceptacion

- [x] Usuarios nuevos siguen redirigidos al tutorial.
- [x] Usuarios existentes pueden abrirlo desde Ajustes.
- [x] El tutorial incluye instrucciones para usar Cachés como app en iPhone y Android.
- [x] Si la app ya esta en standalone, muestra confirmacion en lugar de instrucciones de instalacion.
- [x] En movil usa altura natural, espaciado compacto y controles visibles sin que el tutorial parezca una pantalla gigante.
- [x] El tutorial tiene una accion de cierre; en usuarios nuevos marca onboarding completado para no bloquear la navegacion.
- [x] El final del tutorial guarda `onboarding_completed` mediante `useProfile` y navega a `/settings`.

## Plan tecnico

Ampliar `src/pages/Auth/Onboarding.jsx` con pasos operativos y usar `useStandaloneMode` para adaptar el paso de instalacion.

## Validacion

- E2E de redireccion, paso de instalacion y vuelta a tutorial desde Ajustes.
- Revision responsive en 320, 390, 768 y desktop.

## Resultado

Implementado y cerrado en `RELEASE-0.1.0-beta.21`. `/onboarding` queda como tutorial revisitable, con instrucciones PWA, estado standalone, cierre no restrictivo y version movil compacta para no forzar scroll largo.
