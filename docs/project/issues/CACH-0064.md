# CACH-0064 — Corregir version y copy de consentimiento beta

## Objetivo

Eliminar la referencia obsoleta a beta 8 en consentimiento y tratar `usage_consent_version` como version del texto aceptado, no como numero de release.

## Alcance

- Centralizar `USAGE_CONSENT_VERSION`.
- Sustituir el copy de onboarding y Ajustes por texto de version actual.
- Limpiar `usage_consent_at` y `usage_consent_version` cuando el consentimiento queda desactivado.
- No anadir schema, analitica, PostHog ni tracking.

## Criterios de aceptacion

- [x] No queda `beta-8` ni "beta 8" en el runtime de la app.
- [x] Onboarding y Ajustes usan la misma version de consentimiento.
- [x] El texto visible dice que en esta version no hay analitica real ni eventos de uso.
- [x] Los cambios de perfil siguen pasando por `useProfile`.

## Plan tecnico

Usar constantes compartidas en `src/lib/constants.js` y consumirlas desde `Onboarding` y `Settings`.

- `npm run test`
- `npm run build`

## Resultado

Implementado y cerrado en `RELEASE-0.1.0-beta.21`. El runtime ya no guarda ni muestra `beta-8`; el consentimiento usa `USAGE_CONSENT_VERSION = usage-consent-2026-05` y el copy compartido indica que en esta version no se activa analitica real ni se envian eventos de uso.
