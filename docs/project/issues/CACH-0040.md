# CACH-0040 — [Agents] Separar plan draft de ejecucion mutante

## Objetivo

Separar la planificacion read-only de la ejecucion con efectos locales o remotos, para que `agents:plan` no cree ramas, GitHub Issues ni lance agentes sin intencion explicita.

## Alcance

- Crear `agents:plan:draft` para producir una propuesta local/Product Brain-first.
- Reservar GitHub Issue, rama y lanzamiento de agentes para `agents:plan:execute` o flags equivalentes.
- Alinear `cultura-planner` con la regla de crear primero issue Markdown en `docs/project/issues/`.
- Documentar que GitHub es soporte operativo, no fuente primaria de backlog.

Queda fuera implementar las tareas que salgan del plan o cambiar el proceso de release completo.

## Criterios de aceptacion

- [ ] El modo draft no muta remoto, no crea rama y no lanza agentes de ejecucion.
- [ ] El modo execute declara claramente que puede crear GitHub Issue, rama y runs de agentes.
- [ ] El planner respeta Product Brain-first salvo peticion explicita de GitHub o ejecucion inmediata.
- [ ] El README OpenCode distingue plan, ejecucion local, PR y produccion.
- [ ] La nomenclatura de issues generadas mantiene `CACH-####`.

## Validacion

- Ejecutar `npm run context:check`.
- Ejecutar un plan draft de prueba y confirmar que `git status` solo cambia si el usuario pidio escritura documental.
- Revisar que ningun flujo draft invoca `git push`, GitHub CLI o acciones remotas.

## Resultado

En progreso en `release/0.1.0-beta.11`.
