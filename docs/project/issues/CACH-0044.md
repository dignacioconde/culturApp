# CACH-0044 — [UX] Crear evento desde proyecto con proyecto preseleccionado

## Objetivo

Simplificar el flujo proyecto -> evento para que crear una fecha desde el detalle de un proyecto no exija volver a elegir el proyecto manualmente.

## Alcance

- Anadir una accion clara "Crear evento" desde el detalle de proyecto.
- Abrir el formulario de evento con el proyecto actual preseleccionado.
- Mantener la posibilidad de crear eventos sin proyecto desde el flujo general.
- Conservar hora inicial habitual `08:00`, formato 24h y fin por defecto basado en inicio.

Queda fuera cambiar schema, reglas RLS, calculos de ingresos/gastos o calendario unificado.

## Criterios de aceptacion

- [x] Desde un proyecto, crear evento preselecciona ese proyecto.
- [x] El usuario puede cambiar o quitar el proyecto si el formulario actual lo permite.
- [x] El evento creado aparece vinculado al proyecto y visible en calendario de eventos.
- [x] El flujo general de crear evento sin proyecto no se rompe.
- [x] No se usan controles nativos prohibidos en paginas.

## Validacion

- Ejecutar `npm run lint` y `npm run build`.
- Probar creacion desde detalle de proyecto y desde flujo general.
- Verificar `/calendar/events` y detalle del proyecto tras guardar.
