# CACH-0041 — [UX] Simplificar dashboard movil y estado Ahora

## Objetivo

Hacer que el dashboard movil sea mas escaneable para el uso diario, con una jerarquia clara de lo que ocurre ahora, proximos trabajos y lectura financiera minima.

## Alcance

- Revisar el primer pantallazo movil de `/dashboard`.
- Reducir ruido visual de cards, contadores y bloques secundarios.
- Mejorar el estado "Ahora" para que explique la situacion actual sin ocupar mas espacio del necesario.
- Priorizar accion y orientacion diaria por encima de resumenes decorativos.

Queda fuera cambiar formulas financieras, schema, hooks, RLS o el modelo de datos.

## Criterios de aceptacion

- [x] En 390x844, el primer pantallazo permite entender agenda inmediata sin hacer scroll largo.
- [x] "Ahora" tiene estado vacio, estado con evento actual/proximo y estado sin actividad claramente diferenciados.
- [x] Los KPIs financieros no dominan la vista movil.
- [x] Desktop mantiene densidad y jerarquia profesional.
- [x] Todo texto cabe sin solaparse en 320px de ancho.

## Validacion

- Ejecutar `npm run lint` y `npm run build`.
- Revisar `/dashboard` en 320, 375, 390, 768 y desktop.
- Capturar evidencia visual antes/despues si el flujo de verificacion visual ya existe.

Cerrada por `RELEASE-0.1.0-beta.13`.
