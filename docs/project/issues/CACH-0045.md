# CACH-0045 — [UX] Anadir confirmacion a borrados destructivos

## Objetivo

Evitar perdidas accidentales de proyectos, eventos, ingresos y gastos mediante confirmaciones claras en acciones destructivas.

## Alcance

- Inventariar borrados destructivos visibles en la app.
- Anadir confirmacion antes de borrar entidades principales o movimientos financieros.
- Usar copy breve en espanol de Espana y tuteo.
- Mantener la accion destructiva fuera de superficies principales cuando sea posible.

Queda fuera implementar papelera, undo global, auditoria historica o cambios de schema.

## Criterios de aceptacion

- [x] Borrar proyecto, evento, ingreso o gasto exige confirmacion explicita.
- [x] La confirmacion nombra lo que se va a borrar y el impacto esperado.
- [x] Cancelar no ejecuta mutaciones ni cambia estado visual.
- [x] Confirmar reutiliza las mutaciones existentes sin cambiar contratos de datos.
- [x] La experiencia es usable en 320px y con targets tactiles razonables.
- [x] Los botones inline de ingresos/gastos no borran sin confirmacion previa.
- [x] Confirmar bloquea doble submit mientras se ejecuta el borrado.

## Validacion

- Ejecutar `npm run lint` y `npm run build`.
- Probar cancelar y confirmar en cada entidad cubierta.
- Verificar que no aparecen errores ni estados fantasma tras cancelar.
