# CACH-0095 — Validadores de policies documentales del Product Brain

## Objetivo

Añadir validadores ligeros para que las policies documentales introducidas en Fase 1 sean auditables sin convertir el Product Brain en una arquitectura RAG pesada.

## Alcance

Incluido:

- Validar valores opcionales de `load_policy`, `index_policy` y `context_type` cuando estén presentes.
- Detectar prompts históricos que podrían cargarse por error.
- Detectar documentos `deprecated`, `historical` o `archived` tratados como contexto activo.
- Detectar usos de issues `done` o releases `released` como contexto vivo cuando no haya referencia explícita.
- Mantener los campos nuevos como opcionales; no anotar masivamente documentos existentes.

Fuera de alcance:

- Vector DB, embeddings o chunk index.
- Retrieval profiles ejecutables.
- Renombrado de skills o agentes.
- Cambios en `src/`.

## Criterios de aceptacion

- [x] AC1: `pb:check` detecta valores inválidos de `load_policy`, `index_policy` y `context_type` cuando aparecen.
- [x] AC2: El check distingue documentos excluidos por defecto de documentos realmente rotos, sin fallar por la ausencia de campos opcionales.
- [x] AC3: Los prompts históricos, issues `done`, releases `released` y docs deprecated/historical/archived quedan cubiertos por reglas verificables.
- [x] AC4: No se genera chunk index ni se introduce dependencia de embeddings o servicios externos.
- [x] AC5: `src/` sigue desacoplado del Product Brain.

## Plan tecnico

Extender los checks existentes en `scripts/brain/` o añadir un script local acotado si resulta más claro. Reutilizar el schema v2 y el `document-types-registry` como contrato humano. Evitar migraciones masivas: el primer paso debe validar comportamiento y detectar riesgos, no rellenar metadata en todo el Brain.

## Validacion

- AC1, AC2, AC3: `npm run pb:check -- --strict --json`.
- AC4: revisar diff para confirmar que no hay chunks persistentes, embeddings, vector DB ni servicios externos.
- AC5: `npm run verify:brain`.
- General: `npm run context:check`, `npm run pb:guard` y `git diff --check`.
