# CACH-0085 — Unificar calendarios con visual Lovable sin cambiar comportamiento

## Objetivo

Tokenizar calendarios de eventos/proyectos y la vista anual para que encajen en el sistema Lovable sin reemplazar `react-big-calendar`.

## Alcance

Incluido:

- Superficies, paneles, loading/error/empty states, botones de navegacion, selector de ano y timeline anual.
- CSS RBC solo cuando sea necesario para color, bordes y jerarquia visual.

Fuera de alcance:

- Calendario custom, calendario unificado o redisenar la semana movil.
- Romper altura real, `overflow-x-auto`, `touch-action: pan-x pan-y` o scroll horizontal.

## Criterios de aceptacion

- [x] `/calendar/events` mantiene toolbar, cabeceras, filas/celdas, eventos y vista dia con scroll horizontal.
- [x] `/calendar/projects` mantiene timeline/lista/panel responsive y selector de ano.
- [x] No hay colapso de altura ni overflow incoherente en mobile/tablet/desktop.

## Resultado

Implementada por worker frontend en calendarios: `CalendarEvents`, `CalendarProjects` y `ProjectYearView` migran a tokens Lovable preservando wrappers de altura, scroll horizontal movil y `react-big-calendar`.
