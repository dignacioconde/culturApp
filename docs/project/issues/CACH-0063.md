# CACH-0063 — Unificar BottomActionBar en detalles

## Objetivo

Unificar la barra contextual móvil de detalles de proyecto y evento para que las acciones financieras y de gestión tengan el mismo patrón, sin competir con la navegación inferior global ni tapar contenido.

## Alcance

Incluido:

- Revisar `/events/:id` y `/projects/:id` en móvil.
- Crear o consolidar un patrón compartido de `BottomActionBar` para acciones contextuales.
- Mantener targets táctiles razonables y safe-area correcto.
- Mantener la acción destructiva como secundaria/no dominante y con confirmación.
- Asegurar que el contenido tiene padding inferior suficiente y no queda oculto por la barra.

Fuera de alcance:

- Cambiar fórmulas financieras, KPIs, hooks públicos, RLS, schema o migraciones.
- Cambiar la navegación desktop.
- Reabrir el rediseño completo de detalles financieros.
- Añadir liquidación neta, contratantes o facturación.

## Criterios de aceptacion

- [x] En 320 px y 390 px, `/events/:id` y `/projects/:id` muestran una barra contextual con patrón visual y comportamiento consistente.
- [x] La barra respeta safe-area y mantiene targets táctiles de al menos 40 px.
- [x] El contenido de la página no queda tapado por la barra al hacer scroll hasta el final.
- [x] La navegación inferior global no aparece sobre la barra contextual en detalles.
- [x] La acción destructiva no domina la barra y mantiene confirmación antes de mutar datos.
- [x] Desktop conserva la jerarquía actual o mejora sin perder densidad profesional.

## Plan tecnico

Revisar primero los patrones actuales en `EventDetail` y `ProjectDetail`. Si se extrae componente compartido, mantenerlo en `src/components/ui` o `src/components/layout` según el patrón existente. No tocar helpers financieros ni mutaciones salvo para adaptar callbacks existentes al componente compartido.

## Validacion

- `npm run lint`
- `npm run build`
- Verificación visual en `/events/:id` y `/projects/:id` en 320 px, 390 px, 768 px y desktop.
- Confirmar que editar, cobro rápido, gasto rápido y eliminar siguen abriendo los mismos flujos que antes.

## Resultado

Implementado en `RELEASE-0.1.0-beta.20`: `EventDetail` y `ProjectDetail` usan un `BottomActionBar` compartido para Cobro, Gasto, Editar y Eliminar en móvil. La acción destructiva queda como secundaria visual y mantiene confirmación. Desktop conserva las acciones existentes.
