# CACH-0050 — [Deploy] Tooling local-first de PR release y smoke

## Objetivo

Reducir fricción y riesgo en el camino local hacia PR, release y verificación post-deploy sin automatizar producción ni reglas remotas.

## Alcance

Incluido:

- Scripts local-first para estado de release, sync-check, PR body, ship dry-run, smoke post-deploy y limpieza de ramas locales.
- `verify:ci` y `verify:pr` como preflight reproducible del job `app`.
- `installCommand: "npm ci"` en `vercel.json` manteniendo rewrite SPA.
- Documentación de hardening manual pendiente: PR obligatorio a `main`, revisar auto-revert y no usar scripts para mutaciones remotas sin confirmación.

Fuera de alcance:

- Aplicar GitHub rulesets o branch protection desde script.
- Mutar Vercel remoto, Supabase, Brevo, DNS o producción.
- Automatizar merge a `main` o deploy de producción.

## Criterios de aceptacion

- [x] `npm run release:status` informa release activa, rama, scope, checklist y bloqueos.
- [x] `npm run release:sync-check` falla si la release activa no contiene `main` u `origin/main`.
- [x] `npm run pr:body` genera una descripción PR estándar sin escribir archivos.
- [x] `npm run ship` es dry-run por defecto y no puede mergear ni desplegar producción.
- [x] `npm run smoke:postdeploy -- --url https://culturapp-rho.vercel.app` verifica rutas públicas y SPA fallback.
- [x] `npm run branch:cleanup` no borra ramas sin `--delete-local`.
- [x] Vercel conserva el rewrite SPA y usa `npm ci` como install command versionado.

## Validacion
