# CACH-0078 — Pulir Trabajos y listas con visual Lovable acotada

## Objetivo

Aplicar el lenguaje visual derivado de Lovable a `/work`, `/projects` y `/events` de forma limitada: jerarquia, superficies, tarjetas, badges, filtros, loading y empty states, sin introducir funcionalidades nuevas ni tocar datos.

## Alcance

Incluido:

- Pulido visual de listas/tarjetas existentes en `/work`.
- Pulido visual de las listas existentes de proyectos y eventos.
- Estados de carga, error y vacio con tokens semanticos y utilidades ya disponibles.
- Uso de tokens semanticos de `CACH-0076` para texto, superficies, bordes, acentos y estados.
- Microcopy existente en espanol de Espana, sin renombrar flujos principales.

Fuera de alcance:

- Busqueda nueva, filtro global nuevo o pestana de ano en `/work`.
- Dashboard financiero; queda en `CACH-0079`.
- Cambiar CRUD, rutas funcionales, hooks u origen de datos.
- Command palette, FAB, sheets nuevas o acciones globales.
- Cambios de Supabase, RLS, hooks o migraciones.
- Importar shadcn/Radix/TanStack o componentes Lovable completos.

## Criterios de aceptacion

- [x] `/work` conserva las pestanas y filtros actuales; no aparece busqueda ni pestana de ano nueva.
- [x] `/projects` y `/events` conservan rutas, acciones y comportamiento actual.
- [x] Tarjetas, badges, metadatos y estados vacios se leen bien en mobile y desktop.
- [x] No hay cambios en hooks de datos, contratos Supabase ni componentes de calendario.
- [x] No se anade ninguna dependencia UI nueva.

## Plan tecnico

Trabajar sobre paginas existentes de Work, ProjectList y EventList aplicando clases/tokens ya disponibles. Mantener la logica actual intacta; cualquier necesidad funcional detectada se documenta como follow-up fuera de beta 23.
