# CACH-0080 — Inventariar gaps funcionales Lovable fuera de beta 23

## Objetivo

Documentar los gaps funcionales detectados al comparar el norte Lovable con el alcance real de beta 23, para evitar scope creep: beta 23 incluye tokens y visual acotada, no import de stack ni nuevas funcionalidades de producto.

## Alcance

Incluido:

- Inventario de gaps Lovable que quedan fuera de beta 23.
- Disposicion por gap: no implementar ahora, posible ruta futura y condicion minima para retomarlo.
- Actualizacion del scope de release a "tokens + visual acotada".

Fuera de alcance:

- Implementar cualquiera de los gaps inventariados.
- Importar shadcn/Radix, TanStack Start o componentes completos de `artistic-rhythm`.
- Cambiar Supabase, RLS, hooks, datos o migraciones.
- Cambiar formulas financieras.
- Implementar calendario custom o dark mode.

## Criterios de aceptacion

- [x] El inventario incluye origin-aware back nav, busqueda/tab de ano en Work, calendario custom, command palette, FAB/sheets, dark mode, settings fiscal/preferencias, notificaciones y soporte/privacidad/about.
- [x] Cada gap queda marcado como fuera de beta 23.
- [x] La release `RELEASE-0.1.0-beta.23` refleja alcance de tokens + visual acotada.
- [x] El documento no crea deuda de implementacion escondida ni toca codigo de app.

## Plan tecnico

Actualizar solo Product Brain: issues de beta 23, release activa y current release. Los tableros, indices y digest se regeneran con scripts, no se editan a mano.

## Inventario fuera de beta 23

| Gap Lovable | Disposicion beta 23 | Ruta futura sugerida |
|---|---|---|
| Back nav origin-aware en detalles | Fuera. La beta puede pulir breadcrumbs/estados existentes, pero no cambia semantica de vuelta segun origen. | Slice futura de navegacion bajo core work UX; debe definir origen desde Work, calendario, dashboard y enlaces directos. |
| Busqueda y tab de ano en Work | Fuera. `CACH-0078` no anade busqueda, filtros globales ni tab anual. | `CACH-B0001` o slice hija para Work, con criterios de rendimiento y empty states. |
| Calendario custom | Fuera. Beta 23 no toca calendarios ni reemplaza React Big Calendar o la vista anual actual. | `CACH-B0007`, con criterio UX/capturas y decision sobre calendario unificado. |
| Command palette | Fuera. No se anade busqueda global ni launcher de acciones. | Issue futura si hay IA de acciones frecuentes, permisos y accesibilidad de teclado. |
| FAB y sheets nuevas | Fuera. No se anaden FAB expandible, bottom sheets globales ni nuevas acciones flotantes. | Slice futura mobile-first tras decidir accion primaria por pantalla. |
| Dark mode | Fuera. Los tokens pueden dejar nombres compatibles, pero no se activa tema oscuro. | Decision de sistema visual antes de implementarlo; requiere contraste, persistencia y QA visual completa. |
| Settings fiscal y preferencias ampliadas | Fuera. No se anaden preferencias fiscales ni opciones nuevas de usuario en beta 23. | Issue futura `settings-profile`; cualquier dato nuevo debe usar `useProfile`/usuario autenticado y RLS. |
| Notificaciones | Fuera. No se anaden push, email triggers ni recordatorios. | `CACH-B0008` o slice beta-trust con permisos, privacidad y estados offline claros. |
| Soporte, privacidad y about | Fuera. No se anaden pantallas legales/ayuda/about en esta beta. | Slice futura de confianza beta; separar copy legal, soporte y metadatos de version. |

## Riesgos PB

- Si los gaps se mezclan con las slices visuales, beta 23 deja de ser acotada y se convierte en rediseño funcional.
- Calendario custom y dark mode son tentadores porque Lovable los sugiere visualmente, pero requieren issue propia y validacion especifica.
- Cualquier ajuste de Settings fiscal o preferencias puede tocar datos personales; no entra sin issue de datos/seguridad.
