# CACH-0048 — [Context] Compactar workflow OpenCode

## Objetivo

Reducir carga de contexto y duplicacion en la documentacion OpenCode sin perder las reglas operativas necesarias para agentes.

## Alcance

- Convertir `.opencode/README.md` en quickstart e indice corto.
- Mover secciones extensas, historicas o experimentales a referencias bajo demanda.
- Eliminar duplicacion entre README y `.opencode/AGENT_TASK_TEMPLATE.md`.
- Aclarar que `.memory/` guarda solo conocimiento durable reutilizable, no logs ni historico operativo.
- Documentar allowlist o clasificacion de warnings aceptables de `context:check`.

Queda fuera cambiar app runtime, crear agentes nuevos o redisenar Product Brain completo.

## Criterios de aceptacion

- [ ] `.opencode/README.md` queda como entrada rapida y no como manual completo cargado siempre.
- [ ] La plantilla de tareas vive en un unico sitio canonico.
- [ ] Las notas historicas no se cargan por defecto.
- [ ] `context:check` distingue mejor warnings reales de frases prohibitivas aceptables.
- [ ] Las reglas de memoria no contradicen la politica de carga de contexto.

## Validacion

- Ejecutar `npm run context:check`.
- Ejecutar `npm run pb:check` si se toca Product Brain.
- Revisar que los enlaces internos del README siguen apuntando a archivos existentes.
