# CACH-0048 — [Context] Compactar workflow OpenCode

## Objetivo

Reducir carga de contexto y duplicacion en la documentacion OpenCode sin perder las reglas operativas necesarias para agentes.

## Alcance

- Medir presupuestos de prompts OpenCode y skills portables con `context:check`.
- Clasificar como INFO las frases de carga amplia que sean prohibitivas, condicionales, historicas o referencias concretas.
- Recortar `cultura-lead` para quedar bajo presupuesto.
- Documentar `--concise`/`--caveman` en runners OpenCode sin comprimir zonas sensibles.
- Aclarar que Product Brain y memoria se cargan bajo demanda, no completos por defecto.

Queda fuera cambiar app runtime, crear agentes nuevos, redisenar Product Brain completo o reestructurar todo `.opencode/README.md` en esta ronda.

## Criterios de aceptacion

- [x] `context:check` mide agentes y skills portables.
- [x] Las notas historicas o prohibitivas no se clasifican como warnings reales.
- [x] `context:check` queda con cero warnings y cero errores.
- [x] `cultura-lead` queda bajo presupuesto de contexto.
- [x] Las reglas de memoria no contradicen la politica de carga de contexto.

## Validacion

- Ejecutar `npm run context:check`.
- Ejecutar `npm run pb:check` si se toca Product Brain.
- Revisar que los enlaces internos del README siguen apuntando a archivos existentes.

## Resultado

Compactacion aplicada en la capa de verificadores y prompts cargados por defecto: `context:check` ahora mide skills anidadas, reduce falsos positivos y mantiene cada prompt principal dentro de presupuesto.
