# CACH-0091 — Rehacer visibilidad móvil de calendarios

## Objetivo

Evitar que el calendario móvil pierda visibilidad: Agenda debe combinar calendario y lista accionable, y Plan anual debe priorizar meses/proyectos sin duplicar Gantt y listado anual.

## Alcance

- Agenda móvil con lista del periodo visible.
- Panel seleccionado en flujo normal, no tapando el calendario.
- Plan anual móvil centrado en meses y proyectos del mes seleccionado.
- Mantener altura real de React Big Calendar.

## Criterios de aceptacion

- [x] En móvil se ven calendario y lista útil sin overlay que tape eventos.
- [x] Plan anual no muestra dos representaciones móviles competidoras por defecto.
- [x] Toolbar, cabeceras, filas/celdas y eventos siguen visibles.

## Plan tecnico

Mantener React Big Calendar para Agenda. No reintroducir semana móvil en esta slice. Para Plan anual, conservar timeline desktop y usar una vista móvil por mes seleccionado.

## Validacion

- `npm run lint`
- `npm run build`
- Responsive visual 320, 390, 768 y desktop.

## Resultado

Agenda móvil añade lista accionable del periodo visible y el panel seleccionado vuelve al flujo normal. Plan anual móvil se simplifica en una vista por mes seleccionado.
