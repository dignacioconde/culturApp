# CACH-0047 — [Skills] Actualizar catalogo y symlinks de skills

## Objetivo

Alinear la documentacion y exposicion de skills con el estado real del repositorio para evitar drift entre `.agents/skills`, `.claude/skills` y `docs/agent-skills-strategy.md`.

## Alcance

- Actualizar `docs/agent-skills-strategy.md` con todas las skills actuales.
- Corregir excepciones de `.claude/skills` para que apunten a carpetas canonicas bajo `.agents/skills`.
- Crear o promover `product-brain-orient` si se decide mantener la orientacion Product Brain como skill portable.
- Aclarar triggers entre `cultura-issue-launch`, `cultura-agent-orchestration` y `cultura-release-task-flow`.
- Definir criterio para nombres `*-review` cuando una skill tambien implementa.

Queda fuera reescribir todas las skills o cambiar perfiles OpenCode salvo referencias necesarias.

## Criterios de aceptacion

- [x] El catalogo lista las skills reales y su proposito actual.
- [x] `.claude/skills` no contiene directorios reales duplicados ni symlinks directos a `SKILL.md`.
- [x] Las fronteras entre memoria, Product Brain, issue launch, agentes y release quedan claras.
- [x] `caveman` tiene trigger, procedimiento, excepciones de seguridad y output esperado.
- [x] No se duplican reglas globales largas de `AGENTS.md`.

## Validacion

- Ejecutar `npm run context:check`.
- Ejecutar el nuevo validador de skills si existe; si no, revisar symlinks con `find .claude/skills -maxdepth 1`.
- Ejecutar `npm run pb:check` si se toca Product Brain.

Catalogo y validacion de skills alineados con las 16 skills portables actuales. `caveman` queda integrado como modo de salida conciso en runners OpenCode, con excepciones obligatorias para seguridad, RLS, finanzas, SQL, migraciones, reviews, verificacion y acciones remotas/destructivas.
