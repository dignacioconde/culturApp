# CACH-0053 — [Bug] Mantener foco al escribir feedback

## Objetivo

Corregir el bug por el que el formulario de feedback perdia el foco al escribir la primera letra, cerrando el teclado en movil y haciendo que pareciera que el formulario se cerraba.

## Alcance

- Estabilizar el cierre del modal para que el focus trap no se reinstale en cada render del formulario.
- Estabilizar `lock`/`unlock` del scroll lock compartido.
- Anadir regresion e2e con escritura real en el textarea de feedback.

Fuera de alcance: cambios de Supabase, schema, PostHog o nuevo comportamiento de feedback.

## Criterios de aceptacion

- [x] Al escribir varias letras en el textarea de feedback, el modal sigue visible.
- [x] El textarea conserva el foco despues de escribir.
- [x] El valor escrito no se pierde.
- [x] Hay una prueba e2e que cubre la regresion.

## Validacion

- `npm run lint`
- `npm run test`
- `npm run build`
- `npm run test:e2e`
- `npm run pb:check`

## Resultado

Implementado localmente. El modal ya no reinstala el focus trap al cambiar la identidad de `onClose`, y el scroll lock expone funciones estables. El textarea de feedback conserva el foco mientras se escribe.
