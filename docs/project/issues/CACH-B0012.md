# CACH-B0012 — Perfil público, viralidad y referidos

## Summary

Explorar crecimiento orgánico con perfil público/calendario compartible y sistema de referidos.

## Context

Agrupa #32 y #43.

## Problem

Cada perfil público puede ser una puerta de entrada a Cachés, pero también introduce decisiones de privacidad, disponibilidad y posicionamiento.

## Proposed Solution

- URL pública tipo `caches.es/u/nombreartista`.
- Control granular de visibilidad del calendario.
- Disponibilidad pública opcional.
- Spike de referidos con incentivos.

## Acceptance Criteria

- [ ] Nada se comparte públicamente sin activación explícita.
- [ ] El usuario controla qué datos se muestran.
- [ ] El sistema de referidos tiene hipótesis de MRR o adquisición.
- [ ] Se evalúa después de tener beta funcional.
