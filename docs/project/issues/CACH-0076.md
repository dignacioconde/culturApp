# CACH-0076 — Alinear tokens de diseno con export Lovable

## Objetivo

Extraer del repo Lovable `artistic-rhythm` los tokens y convenciones visuales reutilizables para que Cachés pueda portar clases de diseno como `bg-surface-page`, `text-text-primary`, `border-border-subtle` y `font-display` sin introducir el stack completo de Lovable.

## Alcance

Incluido:

- Anadir aliases semanticos compatibles con el export Lovable dentro de `src/index.css`.
- Mantener estables los tokens actuales de Cachés (`--color-paper`, `--color-ink`, `--color-red`, etc.).
- Exponer tokens Tailwind para fuentes, superficies, texto, acentos, estados y sidebar.
- Anadir utilidades inertes de bajo riesgo (`card-lift`, `skeleton`) para futuros componentes.

Fuera de alcance:

- Importar TanStack Start, Radix/shadcn o componentes completos de `artistic-rhythm`.
- Redisenar pantallas existentes.
- Activar dark mode.
- Cambiar formulas financieras, datos, hooks o Supabase.

## Criterios de aceptacion

- [x] La app sigue compilando con Tailwind v4.
- [x] Las fuentes oficiales siguen siendo `DM Serif Display`, `Instrument Sans` y `DM Mono`.
- [x] Las clases semanticas del export Lovable tienen equivalentes disponibles en Tailwind.
- [x] No se modifica de forma amplia el layout ni la UX actual.

## Plan tecnico

Comparar el export Lovable con `src/index.css` actual y trasladar solo la capa de tokens/aliases, evitando cambios de stack o componentes.
