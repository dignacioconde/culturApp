# CACH-0036 — Profesionalizar flujo de ramas por beta

## Objetivo

Formalizar el flujo de ramas por beta de Cachés: release remota visible, ramas de tarea locales, squash hacia release, PR unica `release/*` -> `main`, tag desde `main` y limpieza de rama remota.

## Alcance

Incluye:

- Actualizar `BRANCHING_STRATEGY`, `RELEASE_PROCESS`, `WORKFLOW` y resúmenes de proceso.
- Refinar ADR-0008 sin crear una ADR nueva.
- Actualizar memoria durable para agentes.
- Probar el propio flujo creando `RELEASE-0.1.0-beta.5`, rama local de tarea y squash hacia release.

Fuera de alcance:

- Cambios de código de app.
- Cambios de CI o branch protection.
- Merge final a `main` y tag de cierre, que quedan para la PR de release.

## Criterios de aceptacion

- [x] `feat/` queda como prefijo preferido y `feature/` solo como legacy.
- [x] Las tareas de beta nacen desde la release activa si pertenecen a ella.
- [x] Una tarea fuera de scope de release no sale de release por defecto.
- [x] Las ramas de tarea son locales por defecto y se integran con squash.
- [x] El cierre de beta se documenta como PR unica `release/*` -> `main`.
- [x] El tag se crea desde `main` actualizado tras mergear la PR.
- [x] Memoria durable queda actualizada para agentes.

## Validacion

- `npm run pb:check`
- `git diff --check`
- `rg` para confirmar que no quedan ejemplos canonicos de merge normal de ramas de tarea ni merge local directo de release a `main`.
