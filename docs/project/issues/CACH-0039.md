# CACH-0039 — [Agents] Respetar permisos reales en lanzadores OpenCode

## Objetivo

Evitar que los lanzadores OpenCode anulen los permisos declarados por cada agente, especialmente en perfiles de review, seguridad, UX y auditoria read-only.

## Alcance

- Revisar `.opencode/scripts/run-agent.mjs`, `.opencode/scripts/run-parallel-agents.mjs` y `.opencode/scripts/run-planner.mjs`.
- Quitar `--dangerously-skip-permissions` como comportamiento por defecto.
- Mapear permisos segun tipo de agente: workers con escritura acotada, reviewers read-only, seguridad read-only y testing con bash controlado.
- Documentar cualquier modo peligroso como opt-in explicito y visible.

Queda fuera redisenar todo el sistema de agentes, cambiar modelos por defecto o ejecutar migraciones de Product Brain.

## Criterios de aceptacion

- [ ] Los agentes con permisos read-only no pueden editar archivos ni ejecutar bash por defecto.
- [ ] Los workers solo pueden escribir cuando el flujo declara ownership concreto.
- [ ] Los scripts fallan con mensaje claro si un perfil pide capacidades no permitidas por el modo elegido.
- [ ] La documentacion OpenCode explica como activar un modo peligroso, si se mantiene, y cuando esta prohibido.
- [ ] `cultura-review`, `cultura-security` y agentes UX quedan protegidos por script, no solo por prompt.

## Validacion

- Ejecutar `npm run context:check`.
- Ejecutar `npm run agents:status`.
- Probar dry-run o ejecucion read-only de un agente de review sin cambios en `git status`.
- Revisar manualmente los comandos efectivos que invocan OpenCode.

## Resultado

En progreso en `release/0.1.0-beta.11`.
