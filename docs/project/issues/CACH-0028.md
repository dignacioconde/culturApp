# CACH-0028 — Corregir sync iCloud y estructura versionada

## Summary

Corregir la implementación inicial del Product Brain para usar el vault real `Product Brain Caches.es`, preservar la estructura completa de carpetas en git y hacer sync manual seguro.

## Problem

La primera versión apuntaba por defecto a un vault legacy y no dejaba versionadas las carpetas vacías de `docs/project/`.

## Proposed Solution

- Usar el vault iCloud real como ruta por defecto.
- Mover el script de sync a `scripts/product-brain-sync.mjs`.
- Mantener estado local de hashes en `.product-brain-sync-state.json`.
- Parar `pb:pull` y `pb:push` si detectan conflictos.
- Añadir índices, plantillas y README/MOC mínimos para que Obsidian navegue bien la estructura.

## Acceptance Criteria

- [x] El vault por defecto es `Product Brain Caches.es`.
- [x] `.obsidian/`, `.DS_Store` y temporales quedan excluidos.
- [x] La estructura completa de `docs/project/` queda versionada.
- [x] `pb:status` muestra estado repo/vault.
- [x] `pb:pull` y `pb:push` paran ante conflictos.
