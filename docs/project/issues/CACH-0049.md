# CACH-0049 — Migrar Product Brain a v2 lean agile para agentes

## Objetivo

Convertir Product Brain en un sistema v2 repo-native, plano, validable y eficiente para humanos, Codex, OpenCode, Claude y skills portables, evitando drift documental, releases activas falsas y cierres sin evidencia.

## Alcance

Incluido:

- Schema v2 plano y compatible con Obsidian para documentos Markdown.
- Migracion v1 -> v2 idempotente de `docs/project/`.
- Issues/backlog con agile ligero: `initiative`, `slice`, `task`, componentes, themes, parents y workflow ejecutable.
- Indices generados por estado, area, release, nivel, componente, theme e iniciativas.
- Digest determinista sin timestamp volatil.
- Checks `pb:ready-check` y `pb:close-check`.
- Orientacion minima para agentes con `pb:orient`.
- Contrato Product Brain v2 en OpenCode agents, runners y skills relevantes.
- CI con validacion de Product Brain, skills y agentes.
- Memoria durable actualizada con reglas Product Brain v2.

Fuera de alcance:

- Cambios de runtime de la app.
- Sincronizacion automatica con Obsidian.
- Crear carpetas Jira o renombrar IDs historicos.
- Cerrar `RELEASE-0.1.0-beta.14`, que sigue reservada para email transaccional definitivo.

## Criterios de aceptacion

- [x] Todos los documentos Product Brain migrados usan `schema_version: 2`.
- [x] No quedan `type/status` top-level en `docs/project/`.
- [x] `pb:index`, `pb:digest` y `pb:migrate-v2` son deterministas y no recrean v1.
- [x] El backlog distingue backlog, ready, in progress, review y done.
- [x] Las issues tienen campos planos para workflow, nivel, prioridad, area, componentes, parent, release y theme.
- [x] `parent` se valida contra initiatives y no permite ciclos.
- [x] `pb:ready-check` y `pb:close-check` existen como guardarrailes.
- [x] Los agentes y skills declaran contexto leido, Product Brain leido/actualizado, validacion PB y feedback/memoria.
- [x] `rg -n "Product Brain|docs/project|CACH-|pb:" src` queda vacio.
- [x] CI incluye checks de app, Product Brain, skills, agentes y whitespace.

## Resultado

Product Brain queda migrado a v2 completo. El repositorio dispone de schema, migrador, orientador, generadores, validadores, checks de cierre y contratos de agente alineados con Product Brain-first.

`CACH-B0019` sigue bloqueada por falta de evidencia historica, como prueba negativa intencional de `pb:close-check`; `CACH-0029` pasa como cierre verificable.

`pb:status` muestra el repo por delante del vault Obsidian con los cambios de migracion y nuevos indices; la sincronizacion con Obsidian se reserva para revision humana posterior.
