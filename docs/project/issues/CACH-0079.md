# CACH-0079 — Pulir Dashboard financiero con visual Lovable acotada

## Objetivo

Aplicar una capa visual acotada al Dashboard financiero usando los tokens de `CACH-0076`, manteniendo los mismos datos, calculos, acciones y modos funcionales.

## Alcance

Incluido:

- Pulido visual de KPIs, panel "Ahora", controles, filas de ingresos pendientes y modal de confirmacion existente en `/dashboard`.
- Estados de carga, error y vacio con tokens semanticos y utilidades ya disponibles.
- Uso de `font-data`/numeros tabulares donde ayude a leer importes y contadores.
- Conservacion de la microcopy y de los modos actuales del Dashboard.

Fuera de alcance:

- Cambiar formulas, agregaciones, acciones de cobro o origen de datos del Dashboard.
- Tocar Work/listas; queda en `CACH-0078`.
- Tocar calendarios, detalles o formularios de proyecto/evento.
- Reemplazar React Big Calendar o crear calendario custom.
- Activar dark mode.
- Cambiar CRUD, datos, hooks, Supabase/RLS o formulas financieras.
- Importar shadcn/Radix/TanStack o componentes Lovable completos.

## Criterios de aceptacion

- [x] `/dashboard` conserva los mismos valores, modos y acciones funcionales.
- [x] KPIs, panel "Ahora", controles e ingresos pendientes usan tokens y jerarquia visual coherentes con `CACH-0076`.
- [x] Importes, contadores y estados se leen bien en mobile y desktop.
- [x] No hay cambios en formulas financieras, hooks de datos ni contratos Supabase.
- [x] No se crea calendario custom, dark mode ni dependencia UI nueva.

## Plan tecnico

Aplicar tokens y clases existentes sobre `src/pages/Dashboard/*`. Evitar tocar helpers de calculo, hooks de datos o acciones; cualquier necesidad funcional detectada se documenta como follow-up fuera de beta 23.
