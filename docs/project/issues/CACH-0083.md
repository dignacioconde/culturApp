# CACH-0083 — Crear primitivas UI para visual total Lovable

## Objetivo

Anadir primitivas visuales internas que reduzcan duplicacion al unificar toda la app con tokens Lovable.

## Alcance

Incluido:

- `Notice`, `EmptyState`, `SearchInput`, `CheckboxField`, `ActionTile`, `MetricTile`, `SectionHeader`, `IconButton` y helpers de variantes si reducen duplicacion real.
- Compatibilidad hacia atras de `Button`, `Card`, `Input`, `Modal` y `Toast`.
- Tokens semanticos para superficies, texto, acentos, estados y datos.

Fuera de alcance:

- Reemplazar librerias UI o importar shadcn/Radix.
- Cambiar comportamiento de paginas, hooks, rutas o datos.

## Criterios de aceptacion

- [x] Las primitivas usan `surface-*`, `text-*`, `accent-*`, `border-subtle`, `danger/success/warning`, `font-display` y `font-data`.
- [x] Las APIs son simples y compatibles con usos progresivos desde las paginas.
- [x] No se cambia comportamiento funcional ni se anaden dependencias.
- [x] `lint`, `build`, `pb:guard` y `git diff --check` pasan.

## Resultado

Implementadas primitivas internas `Notice`, `EmptyState`, `SearchInput`, `CheckboxField`, `ActionTile`, `MetricTile`, `SectionHeader` e `IconButton` sin dependencias nuevas ni cambios funcionales.
